# Glyph Diffuser

Version: 0.17.0  
Project schema: 6

## Purpose

Glyph Diffuser is a deterministic density-based dot renderer. It reuses the existing `GlyphEmitter`, glyph anchors, falloff curves, self/neighbor influence, and `compositeWaveField` source metadata. Unlike Wave Contours, it does not extract isolines or place dots along contour paths.

The pipeline is:

```text
glyph emitter
→ shared radial contribution field
→ bounded jittered candidate grid
→ deterministic probability sampling
→ vector CircleMark dots
```

## Dot model

Candidate spacing comes from Density. Seeded jitter and grain come from Turbulence. The shared emitter contribution supplies radial wave strength; ring contrast weights that signal before deterministic acceptance. Dot radius and opacity combine ring strength, falloff, and seeded grain.

`selfInfluence` affects candidates inside the approximate source-glyph bounds. `neighborInfluence` affects other text and halo candidates. Frequency changes ring spacing, amplitude changes field strength, and halo padding extends the bounded candidate domain.

### Zero-strength semantics

An enabled single emitter with `Strength = 0` is intentionally neutral: it
builds no contributing field source and Glyph Diffuser emits zero marks. In
multiple mode, enabled rows remain visible in row-count diagnostics, but rows
with `weight = 0` are excluded from field composition and produce zero marks.
Other positive-weight rows are unaffected.

Positive strength uses a continuous response curve (`strength^0.85`) against
each emitter's density-owned mark quota. Values around `0.05–0.1` therefore
produce sparse, subtle output; `1` is the normal reference response; values
above `1` increase contribution predictably until the explicit `maxNodes` cap.
Per-emitter quotas prevent a subtle row from taking marks away from another
row unless the shared node cap is reached.

The dev overlay distinguishes enabled configuration from visible contribution
with `contributing`, `zero strength`, per-emitter effective strength, and
marks-per-emitter diagnostics. When every enabled source has zero effective
strength, `fieldState` is `neutral`, not `active`.

## Domains and composition

- **Inside text** samples only the text mask.
- **Emitter halo** samples the bounded emitter radius plus halo padding.
- **Text + halo** accepts either region.
- **Field clipped to text** keeps generated dot centers inside the text mask and retains the normal SVG mask.
- **Field behind text** removes the artwork mask for this renderer only and adds a vector text overlay for readability.

Parsed fonts use glyph paths for the overlay. Native fallback uses SVG `<text>`. The overlay and dots are vector-only.

## Static field, not simulation

Glyph Diffuser is static (`usesTime: false`). It has no temporal feedback, diffusion step, persistent particles, or reaction-diffusion state. The existing animation clock does not run for it.

## Export

Final Artwork exports circles plus an optional SVG text/glyph-path overlay. Editable Text behavior is unchanged. Export contains no canvas, image, PNG, JPEG, or data URL.

## Artboard bounds and edge feather

The preview and exported SVG use a fixed `0 0 1200 720` view box. Emitter field
sampling may extend beyond that artboard, but geometry outside it is intentionally
not visible. Expanding the SVG view box would change the export contract, so large
domains instead use a deterministic 56-unit edge feather before the fixed boundary.
This softens the otherwise abrupt cutoff while preserving fixed-size output.

Renderer diagnostics distinguish the two clipping conditions:

- `artboardBoundsClipped`: an emitter sampling domain exceeds the artboard;
- `artboardEdgeFeather`: `56` when that condition is active, otherwise `0`;
- `maxNodesClipped`: the independent mark budget was reached.

An artboard warning does not mean the marks cap was reached, and a marks-cap warning
does not imply that an emitter domain extends beyond the artboard.

## WebGPU diagnostic relationship

The development WebGPU heatmap previews field state, not final Glyph Diffuser marks.
It can therefore show influence outside the region visible in the fixed SVG artboard,
and it does not reproduce mark sampling, culling, edge feathering, composition, or
the max-node budget. The overlay comparison block reports both field-emitter state
and CPU renderer diagnostics to make that distinction explicit.

WebGPU remains optional and diagnostic-only. Final Artwork SVG is generated by the
CPU/vector renderer and contains vector geometry only.

### Manual QA record

Gate 7.4 was exercised in the Codex in-app Chromium browser on Windows. The WebGPU
overlay mounted with `backend: webgpu-texture`; adapter vendor/device details were
not exposed by the browser QA surface.

Observed:

- **Sonic Diffuser, single mode:** the 370-unit field plus 70-unit halo exceeded
  the vertical artboard bounds. Marks softened before the edge rather than ending
  on a hard horizontal line. The renderer warning and overlay both reported
  `artboard clipped: yes · edge feathered`, while `marks cap: not reached`.
- **Multiple mode:** adding rows live-updated both overlay/runtime counts from one
  to three. Disabling one row updated active/enabled/disabled counts to `2/2/1`.
  Disabling all rows produced `fieldState: neutral` with
  `neutral: no enabled emitter rows`.
- **Mode switch:** switching single → multiple exposed row controls plus clearly
  labeled global shaping controls. Reapplying Sonic Diffuser restored the single
  source and the overlay reported one active runtime/overlay emitter.
- **Overlay lifecycle:** close/reopen restored the current mapping and WebGPU
  backend without stale legend state.
- **Export:** browser status reported `Exported 27.2 KB in 2.3 ms`. Automated
  inspection confirms fixed `viewBox="0 0 1200 720"` and vector-only geometry.
- **Console:** no warning or error entries were captured during the exercised flow.

Not manually exercised in this pass: independent left/right/top/bottom custom-anchor
positions, an approximate/fallback-anchor overlay state, and a visible max-node-cap
case. Those conditions remain covered by targeted diagnostics/unit tests where
applicable. Automated tests deliberately avoid pixel-perfect screenshots because
this repository has no stable visual-diff harness.

## Current limitations

- Source-glyph membership is bounds-based.
- Counter-center selection is heuristic, not topology analysis.
- Native-text cells and anchors are approximate and do not perform shaping.
- Candidate sampling is a jittered grid rather than blue noise.
- Dense output is capped by `maxNodes`; row-major traversal can favor earlier cells when clipped.
- Halo contribution uses the shared emitter formula with an expanded bounded radius.
- The fixed export artboard intentionally clips geometry outside `1200×720`.
